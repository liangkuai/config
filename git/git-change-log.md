# Git change log

以时间为倒序的列表，以记录一个项目中所有版本的显著变动。

### 规则

- 说人话
- 新版本在前，旧版本在后。
- 应包括每个版本的发布日期。

### 变动类型

- `Added` 新添加的功能。
- `Changed` 对现有功能的变更。
- `Deprecated` 已经不建议使用，准备很快移除的功能。
- `Removed` 已经移除的功能。
- `Fixed` 对 bug 的修复。
- `Security` 对安全的改进。
